# Ceph Exporter 中文界面配置说明

## 概述

本项目的所有 Docker 服务已配置为默认使用中文界面，包括：
- ✅ Grafana 仪表盘（中文界面）
- ✅ Prometheus 告警规则（中文描述）
- ✅ Alertmanager 告警通知（中文消息）
- ✅ Kibana 日志分析（中文界面）

## 已配置的服务

### 1. Grafana（监控仪表盘）

**中文配置：**
- 界面语言：简体中文
- Dashboard 标题和面板：全部中文
- 数据源名称：中文标签

**访问地址：** http://localhost:3000
**默认账号：** admin / admin

**中文 Dashboard 包含：**
- 集群概览（健康状态、容量、OSD 数量）
- 容量趋势（总容量、已用容量、使用率）
- OSD 状态（Up/Down 趋势、延迟、使用率）
- 存储池（容量、对象数、读写吞吐量）
- Monitor 状态（Quorum 状态、时钟偏差）
- PG 状态分布
- Exporter 状态

### 2. Prometheus（指标采集）

**中文配置：**
- 告警规则描述：全部中文
- 告警摘要和详情：中文消息

**访问地址：** http://localhost:9090

**中文告警规则包含：**
- 集群健康状态告警（HEALTH_WARN、HEALTH_ERR）
- OSD 告警（宕机、踢出、使用率过高、延迟过高）
- 容量告警（75%、85%、95% 阈值）
- PG 状态告警（非 active+clean 状态）
- Monitor 告警（脱离 quorum、时钟偏差）
- 服务可用性告警（exporter 不可用）

### 3. Alertmanager（告警管理）

**中文配置：**
- 告警路由规则：中文注释
- 告警分组和抑制：中文说明

**访问地址：** http://localhost:9093

**告警级别：**
- warning（警告）：需要关注但不紧急
- critical（严重）：需要尽快处理
- emergency（紧急）：需要立即处理

### 4. Kibana（日志分析）

**中文配置：**
- 界面语言：简体中文

**访问地址：** http://localhost:5601

## 自定义配置

### 修改 Grafana 语言

如需切换回英文，编辑 docker-compose 文件：

```yaml
grafana:
  environment:
    - GF_DEFAULT_LOCALE=en-US  # 改为 en-US
```

### 修改 Prometheus 告警规则

编辑文件：`prometheus/alert_rules.zh-CN.yml`

### 修改 Alertmanager 通知模板

编辑文件：`alertmanager/alertmanager.zh-CN.yml`

## 常见问题

### Q1: Grafana 界面还是英文？

**解决方法：**
1. 清除浏览器缓存
2. 重启 Grafana 容器：`docker-compose restart grafana`
3. 检查环境变量是否正确设置

### Q2: Dashboard 显示 "No Data"？

**解决方法：**
1. 确认 ceph-exporter 正常运行：`docker-compose ps`
2. 检查 Prometheus 是否能采集到数据：访问 http://localhost:9090/targets
3. 确认 Ceph 集群正常运行

### Q3: 告警规则没有生效？

**解决方法：**
1. 检查 Prometheus 配置是否加载成功：http://localhost:9090/config
2. 查看告警规则状态：http://localhost:9090/rules
3. 重新加载配置：`docker-compose restart prometheus`

### Q4: Kibana 界面还是英文？

**解决方法：**
1. 登录 Kibana
2. 点击左上角菜单 → Stack Management → Advanced Settings
3. 搜索 "locale"
4. 将 "Default locale" 改为 "zh-CN"
5. 刷新页面
