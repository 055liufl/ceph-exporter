## 底层实现逻辑

### 核心原理

监控系统的中文化主要涉及三个层次：

```
┌─────────────────────────────────────────────────────────┐
│                    用户界面层                              │
│  - Grafana UI (支持 i18n)                                │
│  - Prometheus UI (不支持 i18n)                           │
│  - Alertmanager UI (不支持 i18n)                         │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    配置文件层                              │
│  - Dashboard JSON (面板标题、图例)                        │
│  - Alert Rules YAML (告警描述)                           │
│  - Prometheus Config (注释说明)                          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    数据标签层                              │
│  - Metric Labels (指标标签)                              │
│  - Annotations (告警注释)                                │
│  - Legend Format (图例格式)                              │
└─────────────────────────────────────────────────────────┘
```

### 实现策略

| 组件 | 中文化方法 | 实现难度 | 效果 |
|------|-----------|---------|------|
| **Grafana** | 环境变量 + Dashboard JSON | ⭐ 简单 | ✅ 完全中文 |
| **Prometheus** | 配置文件中文注释 + 告警规则中文 | ⭐⭐ 中等 | ⚠️ 部分中文 |
| **Alertmanager** | 配置文件中文注释 | ⭐ 简单 | ⚠️ 部分中文 |
| **Kibana** | 环境变量 | ⭐ 简单 | ✅ 完全中文 |

## 技术架构

### 1. Grafana 中文化架构

```
┌──────────────────────────────────────────────────────────┐
│                    Grafana 容器                           │
│                                                           │
│  环境变量: GF_DEFAULT_LOCALE=zh-CN                        │
│      ↓                                                    │
│  加载中文语言包 (内置)                                      │
│      ↓                                                    │
│  渲染中文界面                                              │
│                                                           │
│  ┌─────────────────────────────────────────────┐        │
│  │  Dashboard Provisioning                      │        │
│  │  ↓                                           │        │
│  │  加载 Dashboard JSON                         │        │
│  │  ↓                                           │        │
│  │  解析面板配置:                                │        │
│  │  - title: "集群概览" (中文标题)              │        │
│  │  - legendFormat: "在线" (中文图例)           │        │
│  │  - mappings: "在 Quorum 中" (中文映射)       │        │
│  └─────────────────────────────────────────────┘        │
└──────────────────────────────────────────────────────────┘
```

**关键配置文件：**

1. **docker-compose.yml**
```yaml
grafana:
  environment:
    - GF_DEFAULT_LOCALE=zh-CN  # 设置默认语言为中文
```

2. **Dashboard JSON**
```json
{
  "title": "Ceph 集群监控",  // 中文标题
  "panels": [
    {
      "title": "集群健康状态",  // 中文面板标题
      "targets": [
        {
          "expr": "ceph_health_status",
          "legendFormat": "健康状态"  // 中文图例
        }
      ],
      "fieldConfig": {
        "defaults": {
          "mappings": [
            {
              "options": {
                "0": { "text": "健康", "color": "green" }  // 中文映射
              }
            }
          ]
        }
      }
    }
  ]
}
```

### 2. Prometheus 中文化架构

```
┌──────────────────────────────────────────────────────────┐
│                  Prometheus 容器                          │
│                                                           │
│  配置文件: prometheus.zh-CN.yml                           │
│      ↓                                                    │
│  加载配置 (中文注释，运行时忽略)                            │
│      ↓                                                    │
│  告警规则: alert_rules.zh-CN.yml                          │
│      ↓                                                    │
│  解析告警规则:                                             │
│  ┌─────────────────────────────────────────────┐        │
│  │  - alert: CephClusterWarning                │        │
│  │    annotations:                              │        │
│  │      summary: "Ceph 集群处于 HEALTH_WARN"   │        │
│  │      description: "集群健康状态为 WARN..."   │        │
│  └─────────────────────────────────────────────┘        │
│      ↓                                                    │
│  触发告警时，发送中文描述到 Alertmanager                    │
└──────────────────────────────────────────────────────────┘
```

**关键配置文件：**

1. **alert_rules.zh-CN.yml**
```yaml
groups:
  - name: ceph_cluster_health
    rules:
      - alert: CephClusterWarning
        expr: ceph_health_status == 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Ceph 集群处于 HEALTH_WARN 状态"  # 中文摘要
          description: "集群健康状态为 HEALTH_WARN，已持续超过 5 分钟"  # 中文描述
```

2. **prometheus.zh-CN.yml**
```yaml
# 全局配置
global:
  # 默认采集间隔（每 15 秒从目标拉取一次指标）
  scrape_interval: 15s

# 告警规则文件
rule_files:
  - "/etc/prometheus/alert_rules.yml"  # 映射到中文版本

# 采集目标配置
scrape_configs:
  - job_name: "ceph-exporter"
    static_configs:
      - targets:
          - "ceph-exporter:9128"
        labels:
          service: "ceph-exporter"  # 中文标签
```

### 3. 数据流转过程

```
┌─────────────┐
│ Ceph 集群   │
└──────┬──────┘
       │ 指标数据
       ↓
┌─────────────────────────────────────────────────────────┐
│ ceph-exporter                                            │
│ - 采集 Ceph 指标                                         │
│ - 暴露 Prometheus 格式的指标                              │
│ - 指标名称: ceph_health_status, ceph_cluster_total_bytes │
└──────┬──────────────────────────────────────────────────┘
       │ HTTP /metrics
       ↓
┌─────────────────────────────────────────────────────────┐
│ Prometheus                                               │
│ - 定期采集指标 (scrape_interval: 15s)                    │
│ - 存储时序数据                                           │
│ - 评估告警规则 (中文 annotations)                        │
│ - 触发告警 → Alertmanager (中文描述)                     │
└──────┬──────────────────────────────────────────────────┘
       │ PromQL 查询
       ↓
┌─────────────────────────────────────────────────────────┐
│ Grafana                                                  │
│ - 执行 PromQL 查询                                       │
│ - 渲染 Dashboard (中文标题、图例)                        │
│ - 显示告警 (中文描述)                                    │
│ - 用户界面 (中文)                                        │
└─────────────────────────────────────────────────────────┘
       │
       ↓
┌─────────────┐
│ 用户浏览器   │
│ (中文显示)  │
└─────────────┘
```

## 实施步骤

### 步骤 1：准备中文配置文件

#### 1.1 创建 Prometheus 中文配置

**文件：** `prometheus/prometheus.zh-CN.yml`

```yaml
# =============================================================================
# Prometheus 配置文件 (中文版)
# =============================================================================

global:
  # 默认采集间隔
  scrape_interval: 15s
  # 默认采集超时
  scrape_timeout: 10s
  # 告警规则评估间隔
  evaluation_interval: 15s

# 告警规则文件
rule_files:
  - "/etc/prometheus/alert_rules.yml"

# Alertmanager 配置
alerting:
  alertmanagers:
    - static_configs:
        - targets:
            - "alertmanager:9093"

# 采集目标配置
scrape_configs:
  - job_name: "ceph-exporter"
    scrape_interval: 15s
    static_configs:
      - targets:
          - "ceph-exporter:9128"
        labels:
          service: "ceph-exporter"
          component: "ceph"
```

#### 1.2 创建告警规则中文配置

**文件：** `prometheus/alert_rules.zh-CN.yml`

```yaml
groups:
  - name: ceph_cluster_health
    rules:
      - alert: CephClusterWarning
        expr: ceph_health_status == 1
        for: 5m
        labels:
          severity: warning
          component: cluster
        annotations:
          summary: "Ceph 集群处于 HEALTH_WARN 状态"
          description: "Ceph 集群健康状态为 HEALTH_WARN，已持续超过 5 分钟。请检查集群状态: ceph health detail"

      - alert: CephOSDDown
        expr: ceph_osd_up == 0
        for: 3m
        labels:
          severity: warning
          component: osd
        annotations:
          summary: "OSD {{ $labels.osd }} 已宕机"
          description: "OSD {{ $labels.osd }} 处于 down 状态已超过 3 分钟。请检查该 OSD 所在节点的状态。"
```

#### 1.3 创建 Alertmanager 中文配置

**文件：** `alertmanager/alertmanager.zh-CN.yml`

```yaml
# =============================================================================
# Alertmanager 配置文件 (中文版)
# =============================================================================

global:
  # 告警恢复等待时间
  resolve_timeout: 5m

# 路由规则
route:
  # 分组依据
  group_by: ["alertname", "component"]
  # 首次告警等待时间
  group_wait: 30s
  # 同组告警发送间隔
  group_interval: 5m
  # 重复通知间隔
  repeat_interval: 4h
  # 默认接收者
  receiver: "default-webhook"

# 接收者定义
receivers:
  - name: "default-webhook"
    webhook_configs:
      - url: "http://localhost:5001/webhook/alerts"
        send_resolved: true
```

### 步骤 3：配置 Docker Compose

**文件：** `docker-compose.yml`

```yaml
services:

      - monitor-network

      - monitor-network

    image: prom/alertmanager:v0.27.0

      - ./data/alertmanager:/alertmanager
    command:

networks:
```

### 1. Grafana 中文化详解

#### 1.1 语言设置机制

Grafana 使用环境变量控制界面语言：

```yaml
environment:
  - GF_DEFAULT_LOCALE=zh-CN  # 简体中文
  # 其他可选值:
  # - en-US (英文)
  # - zh-TW (繁体中文)
  # - ja-JP (日文)
  # - ko-KR (韩文)
```

**工作原理：**
1. Grafana 启动时读取 `GF_DEFAULT_LOCALE` 环境变量
2. 加载对应的语言包（内置在 Grafana 中）
3. 渲染界面时使用中文文本

### 2. Prometheus 告警规则中文化详解

#### 2.1 告警规则结构

```yaml
- alert: CephClusterWarning  # 告警名称（建议保持英文，便于代码引用）
  expr: ceph_health_status == 1  # 告警条件
  for: 5m  # 持续时间
  labels:
    severity: warning  # 严重级别
    component: cluster  # 组件标签
  annotations:
    # 关键：使用中文描述
    summary: "Ceph 集群处于 HEALTH_WARN 状态"
    description: "Ceph 集群健康状态为 HEALTH_WARN，已持续超过 5 分钟。请检查集群状态: ceph health detail"
```

#### 2.2 动态变量使用

```yaml
annotations:
  summary: "OSD {{ $labels.osd }} 已宕机"  # 使用变量
  description: "OSD {{ $labels.osd }} 处于 down 状态已超过 3 分钟。当前值: {{ $value }}"
```

#### 2.3 告警分组

```yaml
route:
  group_by: ["alertname", "component"]  # 按告警名称和组件分组
  group_wait: 30s  # 等待 30 秒收集同组告警
  group_interval: 5m  # 同组告警间隔 5 分钟
```

### 3. 配置文件注释中文化

#### 3.1 YAML 注释规范

```yaml
# =============================================================================
# Prometheus 配置文件
# =============================================================================
# 本文件定义 Prometheus 的全局配置、告警规则、告警管理器和采集目标。

scrape_configs:

    static_configs:
      - targets:
```

## 常见问题

### Q1: 为什么 Grafana 界面还是英文？

**可能原因：**
1. 环境变量未正确设置
2. 浏览器缓存
3. Grafana 版本过旧

**解决方法：**
```bash
# 1. 检查环境变量
docker exec grafana env | grep LOCALE

# 2. 重启 Grafana
docker compose restart grafana

# 3. 清除浏览器缓存
```

### Q2: Dashboard 图例显示英文？

**可能原因：**
1. `legendFormat` 使用了英文
2. 颜色覆盖配置中的名称不匹配

**解决方法：**
```json
{
  "targets": [
    {
      "legendFormat": "在线"  // 确保使用中文
    }
  ],
  "fieldConfig": {
    "overrides": [
      {
        "matcher": { "options": "在线" }  // 名称必须匹配
      }
    ]
  }
}
```

### Q3: 告警描述显示英文？

**可能原因：**
1. 使用了英文配置文件
2. 配置文件映射错误

**解决方法：**
```yaml
# docker-compose.yml
volumes:
  - ./prometheus/alert_rules.zh-CN.yml:/etc/prometheus/alert_rules.yml:ro
  # 确保映射到中文版本
```

### Q4: 如何验证配置是否生效？

**验证步骤：**
```bash
# 1. 验证 Prometheus 配置
docker exec prometheus promtool check config /etc/prometheus/prometheus.yml

# 2. 验证告警规则
docker exec prometheus promtool check rules /etc/prometheus/alert_rules.yml

# 3. 查看 Grafana 日志
docker logs grafana | grep locale

# 4. 访问 Grafana API
curl -u admin:admin http://localhost:3000/api/user/preferences
```

### Q5: 如何更新 Dashboard？

**方法一：重启 Grafana**
```bash
docker compose restart grafana
# Grafana 会自动重新加载 Dashboard
```

**方法二：使用 API**
```bash
# 获取 Dashboard JSON
DASHBOARD_JSON=$(cat grafana/dashboards/your-dashboard.json)

# 使用 API 更新
curl -X POST \
  -H "Content-Type: application/json" \
  -u admin:admin \
  http://localhost:3000/api/dashboards/db \
  -d "{\"dashboard\": $DASHBOARD_JSON, \"overwrite\": true}"
```
