## 1. 项目概述

### 1.1 这个项目是做什么的？

**ceph-exporter** 是一个 **Ceph 集群 Prometheus 指标导出器**。简单来说，它做三件事：

1. **连接 Ceph 集群**：通过 RADOS 协议连接到 Ceph 存储集群
2. **采集监控数据**：定期执行 Ceph 命令（如 `ceph status`），获取集群状态
3. **转换并暴露指标**：将数据转换为 Prometheus 格式，通过 HTTP `/metrics` 端点暴露

### 1.2 为什么需要它？

Ceph 集群本身不直接支持 Prometheus 格式。ceph-exporter 充当"翻译器"：

```
Ceph 集群 (JSON) --> ceph-exporter (翻译) --> Prometheus (存储) --> Grafana (展示)
```

### 1.3 核心技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| Go 1.21 | 编程语言 | 高性能编译型语言 |
| go-ceph/rados | Ceph 连接 | Go 的 Ceph RADOS C 库绑定（需要 CGO） |
| prometheus/client_golang | 指标导出 | Prometheus 官方 Go 客户端 |
| logrus | 结构化日志 | 支持 JSON 格式、Hook 机制 |
| lumberjack | 日志轮转 | 按大小/数量/天数自动轮转日志文件 |
| OpenTelemetry | 分布式追踪 | 云原生可观测性标准 |
| Jaeger | 追踪后端 | 追踪数据存储和可视化 |
| Docker | 容器化 | 多阶段构建，生产级镜像 |

### 1.4 项目特点

- **7 个专用采集器**：覆盖集群、存储池、OSD、Monitor、健康状态、MDS、RGW
- **完整可观测性**：指标 + 日志 + 追踪 三位一体
- **插件系统**：支持第三方存储系统监控扩展
- **生产就绪**：优雅关闭、健康检查、TLS 支持、日志轮转

```mermaid
            DS["ClusterStatus 集群状态<br/>PoolStats 存储池统计<br/>OSDStats OSD 统计<br/>MonitorStats Monitor 统计<br/>MDSStats MDS 统计<br/>RGWStats RGW 统计<br/>PGStateEntry PG 状态"]
        end
    end

    CLIENT --> IFACE
    CGO -.->|"实现"| IFACE
    MOCK -.->|"实现"| IFACE
    CLIENT --> LIFE & CMD & DATA
    DATA -->|"返回"| DS

    CEPH["🗄️ Ceph 集群"]
    CGO -->|"RADOS 协议<br/>（C 库调用）"| CEPH
```

**关键设计点：**

1. **接口抽象**：`radosConn` 接口让生产代码和测试代码可以使用不同的实现
2. **线程安全**：`sync.RWMutex` 读写锁，读操作用 `RLock`（允许并发读），写操作用 `Lock`（独占）
3. **超时控制**：`ExecuteCommand` 用 `context.WithTimeout` + goroutine + select 实现超时
4. **CGO 构建标签**：`//go:build cgo` 确保只在启用 CGO 时编译真实连接代码


**每个采集器的工作模式完全相同：**

```
Prometheus 调用 Collect()
    → 创建 10 秒超时上下文
    → 调用 ceph.Client 的 GetXxxStats() 方法
    → 获取 JSON 数据并解析为 Go 结构体
    → 遍历数据，为每条记录创建 prometheus.Metric
    → 通过 channel 发送给 Prometheus
```

**设计要点：**
- `/metrics` 经过追踪中间件，其他端点直接处理

- `/ready` 检查 Ceph 连接（Kubernetes readiness probe）
- 支持 TLS/HTTPS、可配置读写超时

```mermaid
    M->>TR: Shutdown(5s 超时)（defer）
    Note over TR: 刷新待发送的追踪数据<br/>关闭 OTLP 导出器
    TR-->>M: nil

    M->>PLG: Close()（defer）
    Note over PLG: StopAll() 停止所有插件<br/>注销所有采集器<br/>清空映射
    PLG-->>M: nil

    M->>LOG: Close()（defer）
    Note over LOG: 关闭 Logstash Hook<br/>关闭日志文件
    LOG-->>M: nil

    Note over M: 程序退出
```

```go
type Config struct {
```

### 9.2 Ceph 数据结构

```go
// ClusterStatus 集群状态（对应 ceph status -f json）
type ClusterStatus struct {
    Health struct {
        Status string                    // HEALTH_OK / HEALTH_WARN / HEALTH_ERR

                Message string           // 描述信息
            }
        }
    }

        BytesAvail    int64              // 可用空间
        BytesTotal    int64              // 总空间
        ReadBytesSec  int64              // 读取速率（字节/秒）
        WriteBytesSec int64              // 写入速率
        ReadOpPerSec  int64              // 读 IOPS
        WriteOpPerSec int64              // 写 IOPS
        NumObjects    int64              // 对象总数
        PGsByState    []PGStateEntry     // 按状态分组的 PG
    }
    OSDMap struct {
        NumOSDs   int                    // OSD 总数
        NumUpOSDs int                    // Up 状态 OSD 数
        NumInOSDs int                    // In 状态 OSD 数
    }
    MonMap struct {
        NumMons int                      // Monitor 总数
    }
}

// OSDStats OSD 统计（对应 ceph osd df -f json 中的 nodes）
type OSDStats struct {
    ID              int     // OSD ID（如 0, 1, 2）
    Name            string  // OSD 名称（如 "osd.0"）
    Status          string  // "up" 或 "down"
    Reweight        float64 // 权重（1.0=in, 0.0=out）
    TotalBytes      int64   // 总容量（KB）← 注意单位！
    UsedBytes       int64   // 已用容量（KB）
    AvailBytes      int64   // 可用容量（KB）
    Utilization     float64 // 利用率（0-100%）
    PGs             int     // PG 数量
    ApplyLatencyMs  float64 // Apply 延迟（毫秒）
    CommitLatencyMs float64 // Commit 延迟（毫秒）
}
// Up() 方法：Status=="up" 返回 1，否则 0
// In() 方法：Reweight>0 返回 1，否则 0

// PoolStats 存储池统计（对应 ceph osd pool stats -f json）
type PoolStats struct {
    PoolName string  // 池名称（如 rbd, cephfs_data）

    Stats    struct {

        Objects     int64   // 对象数量
        MaxAvail    int64   // 最大可用空间
        BytesUsed   int64   // 已用空间
        PercentUsed float64 // 使用率
    }
    ClientIORate struct {
        ReadBytesSec  int64 // 读取速率
        WriteBytesSec int64 // 写入速率
        ReadOpPerSec  int64 // 读 IOPS
        WriteOpPerSec int64 // 写 IOPS
    }
}

// MonitorStats Monitor 统计（对应 ceph quorum_status -f json）
type MonitorStats struct {
    Name       string  // Monitor 名称
    Rank       int     // 排名
    Address    string  // 地址（IP:端口）
    StoreBytes int64   // 存储大小
    ClockSkew  float64 // 时钟偏差（秒）
    LatencyMs  float64 // 延迟（毫秒）
    InQuorum   bool    // 是否在仲裁中（代码计算，非 JSON 字段）
}
```
