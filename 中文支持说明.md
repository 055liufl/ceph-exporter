# 🌏 中文界面支持

## 快速开始

本项目所有监控服务已完全支持中文界面，让您更轻松地监控 Ceph 集群！

### 一键启动（推荐）

```bash
# 启动完整测试环境（包含 Ceph Demo + 所有监控服务）
./deploy.sh full

# 启动标准部署（连接现有 Ceph 集群）
./deploy.sh standard
# 或
./deploy.sh minimal

# 启动集成测试环境
./deploy.sh test
# 或
./deploy.sh integration

# 查看服务状态
./deploy.sh status

# 查看日志
./deploy.sh logs

# 停止服务
./deploy.sh stop
```

### 访问地址

启动成功后，访问以下地址：

| 服务 | 地址 | 账号 | 说明 |
|------|------|------|------|
| 📊 Grafana | http://localhost:3000 | admin/admin | 中文监控仪表盘 |
| 📈 Prometheus | http://localhost:9090 | - | 指标查询（中文告警） |
| 🔔 Alertmanager | http://localhost:9093 | - | 告警管理（中文配置） |
| 📋 Kibana | http://localhost:5601 | - | 日志分析（中文界面） |
| 🗄️ Ceph Dashboard | http://localhost:8080 | - | Ceph 管理界面 |

## 中文功能特性

### ✅ Grafana 中文仪表盘

- 完整的中文界面
- 所有面板标题和图例都是中文
- 包含以下监控内容：
  - 集群概览（健康状态、容量、OSD 数量）
  - 容量趋势分析
  - OSD 状态监控
  - 存储池统计
  - Monitor 状态
  - PG 状态分布
  - Exporter 性能指标

### ✅ Prometheus 中文告警规则

所有告警规则都使用中文描述，包括：

- **集群健康告警**
  - 集群处于 HEALTH_WARN 状态
  - 集群处于 HEALTH_ERR 状态

- **OSD 告警**
  - OSD 宕机
  - OSD 被踢出集群
  - OSD 使用率过高（> 85%）
  - OSD 延迟过高（> 500ms）
  - 多个 OSD 同时宕机

- **容量告警**
  - 容量使用率 > 75%（警告）
  - 容量使用率 > 85%（严重）
  - 容量使用率 > 95%（紧急）

- **PG 告警**
  - 存在非 active+clean 状态的 PG

- **Monitor 告警**
  - Monitor 不在 quorum 中
  - Monitor 时钟偏差过大

- **服务可用性告警**
  - ceph-exporter 服务不可用

### ✅ Alertmanager 中文配置

- 中文注释的配置文件
- 告警路由规则说明
- 告警分组和抑制规则

### ✅ Kibana 中文界面

- 支持中文界面（需手动设置）
- 日志分析和可视化

所有中文配置文件位于：

```
ceph-exporter/deployments/
├── grafana/
│   ├── dashboards/
│   │   └── ceph-cluster.json              # 中文 Dashboard
│   └── provisioning/
│       ├── datasources/
│       │   └── datasource.yml             # 数据源配置（中文注释）
│       └── dashboards/
│           └── dashboard.yml              # 仪表盘配置（中文注释）
├── prometheus/
│   ├── prometheus.zh-CN.yml               # Prometheus 中文配置
│   └── alert_rules.zh-CN.yml              # 中文告警规则
└── alertmanager/
    └── alertmanager.zh-CN.yml             # Alertmanager 中文配置
```

## Docker Compose 配置

所有 docker-compose 文件已更新为默认使用中文配置：

1. **docker-compose.yml** - 标准部署
2. **docker-compose-lightweight-full.yml** - 完整测试环境
3. **docker-compose-integration-test.yml** - 集成测试

关键环境变量：

```yaml
grafana:
  environment:
    - GF_DEFAULT_LOCALE=zh-CN  # Grafana 中文界面

kibana:
  environment:
    - I18N_LOCALE=zh-CN        # Kibana 中文界面
```

## 验证中文界面

### 1. Grafana

1. 访问 http://localhost:3000
2. 登录（admin/admin）
3. 左侧菜单应显示中文
4. 打开 "Ceph 集群监控" 仪表盘
5. 所有面板标题应为中文

### 2. Prometheus

1. 访问 http://localhost:9090
2. 点击 "Alerts" 查看告警规则
3. 告警描述应为中文

### 3. Kibana

1. 访问 http://localhost:5601
2. 如果界面是英文，进入 Stack Management → Advanced Settings
3. 搜索 "locale"，设置为 "zh-CN"
4. 刷新页面

### 4. 自动验证

使用验证脚本自动检查所有配置：

```bash
./verify.sh
```

该脚本会检查：
- 中文配置文件是否存在
- Docker Compose 中文配置是否正确
- 运行中服务的中文配置
- 容器状态和健康检查
- 服务端点可访问性
- Prometheus 和 Grafana 配置
- 资源使用情况

## 切换回英文

如需切换回英文界面，编辑 docker-compose 文件：

```yaml
grafana:
  environment:
    - GF_DEFAULT_LOCALE=en-US  # 改为英文

# 同时修改配置文件引用
volumes:
  - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml:ro  # 使用英文配置
  - ./prometheus/alert_rules.yml:/etc/prometheus/alert_rules.yml:ro
  - ./alertmanager/alertmanager.yml:/etc/alertmanager/alertmanager.yml:ro
```

然后重启服务：

```bash
docker-compose restart
```

## 常见问题

### Q: Grafana 界面还是英文？

**A:** 尝试以下方法：
1. 清除浏览器缓存
2. 重启 Grafana：`./deploy.sh stop && ./deploy.sh full`
3. 检查环境变量：`docker exec grafana env | grep LOCALE`

### Q: Dashboard 显示 "No Data"？

**A:** 检查以下内容：
1. ceph-exporter 是否正常运行：`./deploy.sh status`
2. Prometheus 是否能采集数据：访问 http://localhost:9090/targets
3. Ceph 集群是否正常

### Q: 告警规则没有生效？

**A:** 验证配置：
```bash
# 使用部署脚本验证
./deploy.sh verify

# 或手动检查
docker exec prometheus promtool check config /etc/prometheus/prometheus.yml
docker exec prometheus promtool check rules /etc/prometheus/alert_rules.yml

# 重新启动
./deploy.sh stop
./deploy.sh full
```
